## 📖 项目背景
在当前的招聘流程中，HR 或面试官需要花费大量时间筛选海量简历。本项目旨在通过自动化工具自动解析候选人 PDF 简历，利用 AI 大语言模型（DeepSeek）提取核心结构化信息，并根据岗位需求 (JD) 智能计算匹配度评分，从而大幅提升招聘初筛效率。

## 🏗️ 架构设计与技术选型

本项目采用前后端分离的 Monorepo 架构，并通过 Docker 实现了完全的容器化环境隔离：

### 后端 (Backend) - 遵循 RESTful API 规范
- **框架**: `FastAPI` (原生支持异步，极速响应，易于平滑迁移至 Serverless 架构)
- **AI 引擎**: `DeepSeek API` (兼容 OpenAI SDK，强大的结构化数据抽取与推理能力)
- **PDF 解析**: `PyMuPDF (fitz)` (高效精准提取多页复杂排版的文本)
- **缓存层**: `Redis` (利用 MD5 计算文件与岗位需求的唯一 Hash，实现分析结果秒级响应，包含**优雅降级策略**)
- **数据验证**: `Pydantic` (严格约束出入参 JSON 结构，并作为提示词结构化输出的参考基准)

### 前端 (Frontend)
- **框架**: `Vue 3` + `Vite` + `<script setup>` (现代化极速前端构建栈)
- **UI 组件库**: `Element Plus` (构建企业级中后台管理交互质感)
- **网络代理**: 采用 `Nginx` 统一暴露 80 端口，配置反向代理解决前后端跨域与 SPA 路由问题。

## ✨ 核心功能模块

1. **简历上传与解析**: 支持单文件 PDF 拖拽上传，精准提取文本并进行清洗去重（去除冗余空白与换行）。
2. **关键信息提取**: 深度融合提示词工程（Prompt Engineering），精准提取候选人姓名、联系方式、教育背景及高阶核心项目经历。
3. **简历评分与匹配**: 根据输入的招聘岗位需求，多维度（综合评分、技能匹配率）进行评估，并由 AI 输出具体的诊断建议。
4. **结果缓存机制 (加分项)**: 引入 Redis，通过对 `PDF字节流 + JD特征` 计算 Hash Key。缓存命中时瞬间返回结果，有效规避大模型重复开销。
